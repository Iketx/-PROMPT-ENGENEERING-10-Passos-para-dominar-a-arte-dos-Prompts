

> [!tldr] 
> Aprenda como prevenir, detectar e lidar com erros da IA facilmente. Domine as técnicas de consistência da precisão e recuperação de erros.

---

## 1. Compreendendo erros da IA

A IA pode cometer inumeros erros. Compreende-los pode nos ajudar a preveni-los e lidar melhor com eles.

> [!info] Os Erros mais comuns:
>* Alucinação;
>* Confusão de contexto;
>* Inconsistencia na formatação;
>* Erros lógicos;
>* Respostas incompletas.

## 2. Técnicas de prevenção de erros

A melhor forma de lidar com erros é prevenir que eles aconteçam. Segue abaixo como o fazer:

**Prompt basico (Propenso a erros):**
```
Escreva um sumario sobre a performance da empresa no ultimo ano.
```

**Prompt com prevenção de erros:**
```
Forneça um sumario sobre a performance da empresa em 2024 usando essas restrições:

ESCOPO:
- Foque apenas em verificar as métricas financeiras;
- Inclua os dados específicos de cada trimestre.
- Faça referencia aos numeros apurados neste ano

VALIDAÇÃO REQUERIDA:
- Se um numero é estimado, marque como est.
- Se um dado é incompleto, anote os períodos que faltam;
- Para as projeções, rotule de forma clara como "projeção"

FORMATO:
Metrica: [Receita/Lucro/Crescimento]
Dados do 1T até o 4T: [Numeros trimestráis]
Evolução ano a ano: [%]
Status: [Verificado/Estimado/Projetado]
```

> [!NOTE] Por que isso funciona melhor?
> * Porque claramente separa dados verificados e dados estimados.
> * Porque previne a mistura de números atuais e projetados.
> * Permite observar qualquer gap de dados de forma implícita.
> * Garante um relatório transparente.

## 3. Técnicas de auto-verificação.

Faça com que a IA cheque seu próprio trabalho e mostre potenciais problemas

**Solicitação de análise básica:**
```
Analise os dados comerciais e mostre as tendencias.
```

**Solicitação de análise auto verificante:**
```
Analise estes dados de vendas usando o seguinte framework:

1. Checagem de dados:
	- Confirme se os dados estão completos;
	- Verifique qualquer gap ou anomalia;
	- Marque padrões suspeitos.

2. Passos para a anãlise:
	- Demonstre os calculos;
	- Explique a metodologia;
	- Liste as suposições feitas

3. Verificação de resultados:
	- Verifique os calculos;
	- Os compare contra benchmarks;
	- Marque as incoerencias descobertas.

4. Nível de Confiança
   - Alto: Dados claros, cálculos verificados
   - Médio: Algumas premissas feitas
   - Baixo: Incerteza significativa

FORMATAR RESULTADOS COMO:
Status dos Dados Brutos: [Completo/Incompleto]
Método de Análise: [Descrição]
Resultados: [Lista]
Confiança: [Nível]
Notas de Verificação: [Quaisquer preocupações]
```

## 4. Padrões de detecção de erros

Aprenda a detectar potenciais erros antes que eles se tornem uma catastrofe.

#### ⚠️ Checagem de inconsistencia
```
VERIFIQUE A CONSISTENCIA:
1. Checagem de números:
	- Os números somam?
	- As porcentagens são lógicas?
	- As tendencias são consistentes?


2. Testes logicos:
	- As conclusões são apoiadas nos dados?
	- Existem contradições?
	- O raciocínio é sólido?
	  

3. Verificações de Contexto
   - Isso corresponde aos fatos conhecidos?
   - As referências são precisas?
   - Existe no contexto uma lógica temporal coerente?
```

#### ⚠️ Prevenção a alucinação:
```
VERIFICAÇÃO DE FATOS REQUERIDA:
- Marque claramente o conteúdo especulativo
- Inclua níveis de confiança
- Separe fatos de interpretações
- Anote as fontes de informação
- Sinalize as suposições de forma clara
```

## 5. Estratégias de recuperação de erros

Quando você identifica um erro na resposta da IA, aqui está como você deve proceder para corrigir:

**Prompt de correção de erros:**
```
Em sua resposta anterior sobre o [tópico], há um erro:
[Copie o erro aqui]

Portanto:
1. Corrija o erro especificado
2. Explique o porque do erro
3. Disponha a informação correta
4. Aponte se esse erro afeta outras partes de sua resposta.
```

Exemplo preenchido:
```
Em sua resposta anterior sobre nossa análise de vendas do T4,
você afirmou que nosso crescimento foi de 25% ao comparar o T4 com o T3.
Isso está incorreto, de acordo com nossos relatórios financeiros.

Por favor:
1. Corrija este erro específico
2. Explique por que estava incorreto
3. Forneça o número correto de crescimento do T4 vs T3
4. Observe se isso afeta suas outras conclusões
```

## 6. Prevenção de erros de Formatação.

Previna erros relacionados a formatação, com modelos claros:

**Imposição de modelo:**
```
REQUISITOS DE SAÍDA:
1. Estrutura
   [ ] Cabeçalhos de seção presentes
   [ ] Níveis de aninhamento corretos
   [ ] Formatação consistente

2. Verificações de Conteúdo
   [ ] Todas as seções concluídas
   [ ] Elementos necessários presentes
   [ ] Sem texto de espaço reservado

3. Validação de Formato
   [ ] Uso correto de marcadores
   [ ] Numeração adequada
   [ ] Espaçamento consistente
```

## 7. Prevenção de erros Lógicos.

Veja como solicitar a IA que verifique o seu próprio raciocínio logico[^1].
```
Antes de fornecer sua resposta final sobre [tópico], verifique seu raciocínio usando estas etapas:

1. Verifique seu ponto de partida
   "Baseei minha análise nessas premissas..."
   "Usei essas definições..."
   "Minhas condições iniciais foram..."

2. Verifique as etapas do seu raciocínio
   "Aqui está como cheguei à minha conclusão..."
   "As etapas principais do meu raciocínio foram..."
   "Passei de A para B porque..."

3. Valide suas conclusões
   "Minha conclusão decorre das etapas porque..."
   "Considerei essas alternativas..."
   "Estas são as limitações da minha análise..."
```
**Exemplo:**
```
Antes de fornecer sua recomendação final para nossa estratégia de marketing, por favor:

1. Declare suas premissas iniciais sobre:
   - Nosso mercado-alvo
   - Nosso orçamento
   - Nosso cronograma

2. Mostre como você chegou à sua recomendação:
   - Explicando cada etapa
   - Mostrando por que cada decisão leva à próxima
   - Destacando os principais pontos de virada

3. Valide sua recomendação final:
   - Conectando-a de volta aos nossos objetivos
   - Observando quaisquer limitações
   - Mencionando abordagens alternativas consideradas
```
[^1]: Em modelos de IA do tipo "reasonning" muitas vezes o raciocínio estará implícito na resposta. Ex. Deepseek-R1, Gemini Flash Thinking e Botão Refletir do Chat-GPT.

## 8. Guia de implementação.

1. **Sempre inclua etapas de verificação**
    
    - Incorpore verificações nos prompts iniciais
        
    - Solicite marcação explícita de incerteza
        
    - Inclua níveis de confiança
        
2. **Use categorias de erros claras**
    
    - Erros factuais
        
    - Erros lógicos
        
    - Erros de formato
        
    - Erros de conclusão
        
3. **Mantenha logs de erros**
    
    - Acompanhe problemas comuns
        
    - Documente correções bem-sucedidas
        
    - Crie estratégias de prevenção
