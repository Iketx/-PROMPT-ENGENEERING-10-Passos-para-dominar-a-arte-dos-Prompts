## 1. Compreendendo a geração de conteúdo

Os *prompts* para geração de conteúdo precisam de uma estrutura clara e parâmetros específicos afim de gerar saídas de alta qualidade. Por isso, é importante assimilar que os diversos tipos de conteúdo existentes irão demandar diferentes abordagens.

> [!question] Por que um *prompt* com uma estrutura clara importa?
> * Garante consistencia na qualidade
> * Mantem o tom e estilo
> * Produz conteúdo relevante
> * Permite um processo eficiente de revisão
> * Facilita o ajuste fino, alinhamento de marca ou proposito.

## 2. Controle estrutural do conteúdo

**Abordagem simplista:**
```
Escreva um post sobre dicas de produtividade.
```

**Abordagem estruturada:**
```
Crie um post para um blog com as seguintes específicações:

FORMATAÇÃO:
	1. Titulo [Amigável ao SEO].
	2. Introdução [100 palavras]:
	   - Frase de efeito,
	   - Contextualização,
	   - Introdução dos topicos principais.
	3. Desenvolvimento:
	   - 3 ou 4 pontos-chave,
	   - Para cada ponto-chave: [subtítulo + 200 palavras],
	   - Inclua exemplos reais,
	   - Inclua dicas praticas.
	4. Conclusão [100 palavras]:
	   - Sumarize os ponto-chave,
	   - CTA [Chamada para a ação]
     
ESTILO:
    - Tom: Profissional e amigavel.
    - Nivél: Audiencia com familiaridade intermediaria.
    - Voz: Ativa e engajadora.
    - Diagramação: Fácil de ler e com subtítulos.

INCLUSOS:
	- Exemplos práticos,
	- Dados estatísticos e fontes,
	- Conclúsões práticas,
	- Analogias didaticas.
```

> [!success] Porque essa abordagem é melhor?
> * Clareza estrutural,
> * Separação em seções,
> * Específicação de requerimentos,
> * Guia de estilização.

## 3. Compreendendo os *frameworks* de conteúdo aplicados a diferentes propósitos.

**Relatório de negocio:**
```
CRIE: [tipo do documento]
FINALIDADE: [o objetivo do documento]

ESTRUTURA:
	1. Sumário executivo:
	   - Pontos-chave,
	   - Descobertas principais,
       - Recomendações.
    2. Conteúdo principal:
       - Background,
       - Análise,
       - Dados relevantes.
    3. Conclusões:
       - Pontos de acão,
       - Linha do tempo,
       - Proximos passos.

GUIA DE ESTILO:
	"Use um tom profissional, claro, conciso, orientado aos Dados e a ação."
	
FORMATAÇÃO:
	- Use tópicos para listas,
	- Inclua métricas relevantes,
	- Adicione gráficos quando relevante,
	- Inclua recomendações.
```

**Manual técnico ou documentação:**
```
CRIE: [manual/documentação]
TIPO: [guia do usuário/documentação de api/tutorial]

ESTRUTURA:
	1. Visáo geral:
	   - Propósito,
	   - Pré requisitos,
	   - Conceitos-chave.
	1. Passo-a-passo:
	   - Instruções claras,
	   - Exemplos de codigo ou similares,
       - Problemas comuns,
       - Melhores práticas.
    1. Referencias:
       - Detalhes técnicos,
       - Parametros,
       - Exemplos.

ESTILO:
	- Atente a precisão técnica,
	- Use explicações claras,
	- Seja consistente com os termos técnicos,
	- Forneça exemplos práticos suficientes.
```
